# 生成内容质量对比报告标准

当报告要比较新旧版本、模型、prompt、渲染链路、解析本、讲解稿、总结、UI 文案或其他生成内容质量时，读取本 reference。

## 核心问题

报告必须回答：

> 新版本是否在正确维度上足够好，并且风险可接受，因此可以替代旧版、灰度或继续验证？

平均分不能单独支撑结论。

## 必备证据

1. **质量维度**
   - 明确定义本任务中的“质量”是什么。
   - 例如：正确性、完整性、结构性、有用性、可读性、一致性、展示质量、用户适配度。

2. **主决策指标**
   - winner 数、通过率、人工偏好、绝对质量阈值或加权分。
   - 说明该指标为什么能回答主决策。

3. **分布信息**
   - 均分。
   - winner 数。
   - 持平数。
   - 极端低分 case。
   - 回归 case。

4. **评测工具可靠性**
   - 如果使用 LLM-as-judge，必须说明是否经过人工校准。
   - 如果没有人工校准，结论只能支持观察或灰度建议。

5. **分桶分析**
   - 新版在哪些场景胜出？
   - 旧版在哪些场景胜出？
   - 哪些用户可见失败最重要？

6. **决策边界**
   - 全量替代、灰度、局部使用、继续验证或拒绝。

## 常见问题

| 问题 | 判断信号 | 纠偏方向 |
| --- | --- | --- |
| 均分越级 | 新版均分更高，所以说新版全面更好 | 补 winner 数、回归 case 和低分 case |
| 忽略评测偏差 | LLM judge 未校准就作为最终判断 | 补人工盲评或降级结论 |
| 质量维度混淆 | 结构变好，但正确性变差 | 拆分维度并回到产品优先级 |
| 极端失败被隐藏 | 平均表现好，但存在严重坏例 | 补低分和回归 case 复核 |
| 替代结论过强 | 证据只支持灰度，却写成全量替换 | 让决策强度匹配证据强度 |

## 推荐结构

1. 对比问题与决策目标。
2. 对比版本与样本口径。
3. 质量维度与评分 rubric。
4. 评测工具可靠性 / 校准情况。
5. 主结果：winner 数或主决策指标。
6. 分数分布与分桶分析。
7. 回归 case 和极端失败 case。
8. 解释边界。
9. 决策：替代 / 灰度 / 局部使用 / 继续验证。

## 示例

通用规则：

> 新版本均分更高，并不一定适合全量替代。若它在更多 head-to-head case 中失败、存在严重回归，或评测器未经人工校准，就只能支持灰度观察或继续验证。

历史示例：

在对比新旧两个生成内容版本时，新版部分维度均分更高，但旧版 head-to-head 胜出更多。这类证据可以支持灰度观察，不能自动支持全量替代。
